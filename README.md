# 🌊 ReefWatch — Global Marine Biodiversity & Protection Map

An interactive Leaflet map combining GEE tile layers, WDPA protected area polygons, OBIS live biodiversity data, and a traffic-light risk analysis system.

---

## 📁 Project Structure

```
reef-watch/
├── index.html                  ← Main interactive map (deploy this)
├── assets/
│   ├── tile_urls.json          ← Generated by GEE script (gitignored until regenerated)
│   ├── mpa_strict.geojson      ← IUCN Ia/Ib/II polygons (from GEE export)
│   ├── mpa_managed.geojson     ← IUCN III–VI polygons (from GEE export)
│   └── obis_hex_grid.geojson   ← Pre-computed biodiversity hex grid
└── scripts/
    ├── gee_export_layers.py    ← Run in Google Colab — exports GEE tiles
    ├── obis_grid.py            ← Run in Google Colab — builds OBIS hex grid
    └── colab_setup.py          ← Colab notebook setup reference
```

## 🚀 Setup in 5 Steps

### Step 1 — Create GitHub Repository

```bash
# On your machine
git init reef-watch
cd reef-watch
git remote add origin https://github.com/YOUR_USERNAME/reef-watch.git
mkdir assets scripts
```

Copy all project files into this directory.

### Step 2 — Enable GitHub Pages

1. Push to GitHub: `git add . && git commit -m "init" && git push origin main`
2. Go to **Settings → Pages**
3. Set source: **Deploy from branch → main → / (root)**
4. Your map will be live at: `https://YOUR_USERNAME.github.io/reef-watch/`

### Step 3 — Run GEE Export in Google Colab

Open a new Colab notebook and paste (or upload `scripts/gee_export_layers.py`):

```python
# Cell 1 — Install
!pip install earthengine-api geemap h3

# Cell 2 — Auth
import ee
ee.Authenticate()
ee.Initialize(project='YOUR_PROJECT_ID')

# Cell 3 — Run export
%run scripts/gee_export_layers.py
```

This generates:
- `assets/tile_urls.json` — GEE tile URL endpoints (valid ~2 hours, refresh periodically)
- `assets/mpa_strict.geojson`
- `assets/mpa_managed.geojson`

### Step 4 — Run OBIS Grid (Optional, ~10 min)

```python
%run scripts/obis_grid.py
```

Generates `assets/obis_hex_grid.geojson` — pre-computed biodiversity hotspots.

> **Note**: OBIS live queries still work on click without this file. The hex grid just adds the background biodiversity layer.

### Step 5 — Push Assets & Deploy

```bash
git add assets/
git commit -m "Add GEE tiles and OBIS grid"
git push origin main
```

GitHub Pages auto-deploys within ~30 seconds.

## 🎯 Thresholds Explained

### Traffic Light (Risk Analysis)

| Colour | Condition | Scientific Basis |
|--------|-----------|-----------------|
| 🟢 **Green** | MPA present AND no bleaching AND DHW < 0 | Area is legally protected with no current thermal stress |
| 🟡 **Yellow** | MPA present AND (bleaching present OR DHW ≥ 4) | Protected but under thermal stress — requires monitoring |
| 🔴 **Red** (pulsing) | No MPA AND bleaching present | Unprotected reef under active bleaching — highest risk |

> **Note**: Traffic light uses green/yellow/red, which is deliberately distinct from the SST blue→orange→red palette and DHW white→dark-red palette to avoid confusion.

### Sea Surface Temperature

| Value | Meaning |
|-------|---------|
| 20°C (blue) | Cool — no thermal stress |
| 26°C (pale) | Typical tropical baseline |
| 29–32°C (orange/red) | Above thermal stress threshold |

### Degree Heating Weeks (DHW)

| DHW | NOAA Level | Implication |
|-----|-----------|-------------|
| 0 | Watch | Bleaching possible |
| ≥ 4 | Alert Level 1 | Significant bleaching expected |
| ≥ 8 | Alert Level 2 | Severe bleaching & mortality likely |

Calculated as sum of weekly positive SST anomalies (°C above climatological MMM) divided by 7, over the preceding 8–12 weeks. Baseline: 1985–2013 NOAA OISST climatology.

### Allen Coral Atlas Bleaching Classes

| Class | Meaning |
|-------|---------|
| 1 | No bleaching |
| 2 | Suspect/minor |
| 3 | Moderate bleaching (threshold used) |
| 4 | Severe bleaching |

### OBIS Biodiversity

| Threshold | Meaning |
|-----------|---------|
| ≥ 10 species / H3 hex cell | "Significant" biodiversity zone — highlighted on map |
| H3 resolution 4 | ≈ 1,700 km² per hexagon — regional scale |
| Colour = teal | Inside MPA boundary |
| Colour = orange | Outside any MPA — unprotected diversity hotspot |

## 🔄 Refreshing GEE Tile URLs

GEE tile URLs expire after ~2 hours. To refresh:

1. Re-run `gee_export_layers.py` in Colab
2. Copy new `tile_urls.json` to `assets/`
3. Push to GitHub

For a persistent deployment, consider:
- Hosting a small Flask/FastAPI server that generates fresh GEE tiles on demand
- Using GEE's `ee.data.getMapId()` via a Cloud Function
- Upgrading to a `geemap` + `solara` app on Hugging Face Spaces

## 🛠 Troubleshooting

**Map shows "DEMO MODE"**: `tile_urls.json` not found. Run GEE export first.

**GEE authentication fails**: Run `ee.Authenticate()` and follow browser prompts. Ensure your GEE project has Earth Engine API enabled.

**OBIS panel shows "API unavailable"**: OBIS API is rate-limited. Wait 30s and retry, or check https://status.obis.org.

**Tiles not showing after push**: Check tile URL expiry. GEE URLs are signed and expire in ~2h. Regenerate.
